# 全局主题与顶部图标设计

## 目标

为 UpTimer Web 与 ArkWeb 端增加全局浅色、深色、跟随系统三态主题，并简化顶部导航为图标按钮。

## 顶部导航

- 删除左上角菜单（三条杠）图标及其占位区域。
- “活动管理”保留原有链接行为，改为活动管理 SVG 图标按钮。
- “退出”保留原有退出行为，改为退出 SVG 图标按钮。
- 图标按钮必须提供 `aria-label` 与 `title`，保证无文字时仍可访问和识别。
- 新增主题切换按钮，按以下顺序循环：浅色 → 深色 → 跟随系统 → 浅色。
- 主题按钮图标分别使用太阳、月亮、电脑图标，并随当前主题状态变化。

## 主题状态

主题设置使用三个值：`light`、`dark`、`system`。

- 使用 `localStorage` 持久化用户选择。
- 没有已保存设置时默认为 `system`。
- `system` 模式通过 `matchMedia('(prefers-color-scheme: dark)')` 应用系统主题，并监听系统主题变化。
- `light` 和 `dark` 模式不受系统主题变化影响。
- 首次加载时避免因客户端主题初始化导致明显闪烁；服务端渲染内容保持可读。

## 全局样式

- 在根节点标记当前有效主题，统一由 CSS 变量控制背景、卡片、边框、主文字、弱文字、输入控件、弹窗、底部导航和按钮颜色。
- 深色主题保持现有蓝色主色、绿色收益色、橙色损失色和红色危险色的语义，但降低高亮背景和阴影对比度。
- 页面、表单、弹窗、选择器、时间轴、统计图表和底部导航均必须适配深色背景，不能出现白色卡片或黑色文字残留。
- 浅色主题保持现有视觉不变。
- ArkWeb 端复用同一套网页主题逻辑，不新增鸿蒙专用主题实现。

## 验收标准

1. 顶部左侧不再显示三条杠图标。
2. 活动管理、退出按钮只显示对应 SVG 图标，点击行为不变。
3. 主题按钮连续点击三次能依次切换浅色、深色、跟随系统，图标同步变化。
4. 刷新页面后主题选择保持；重新打开 ArkWeb 后主题选择保持。
5. 跟随系统模式下切换系统浅色/深色，页面自动更新。
6. 所有主要页面和弹窗在深色主题下文字、边框、背景和状态颜色清晰可读。
7. `npm run build` 通过，主题逻辑具备自动化测试覆盖。
